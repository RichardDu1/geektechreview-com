## Overview

Real-time progress monitoring on construction sites is critical for staying on schedule and budget. By combining drone imagery with AI change detection, you can automatically compare daily orthomosaic maps and highlight differences—foundation pours, steel erection, equipment movement, etc. This guide walks you through a complete B2B workflow using industry-standard tools.

## Prerequisites

- A drone capable of capturing high-resolution images with GPS geotagging (e.g., DJI Mavic 3 Enterprise)
- A drone flight planning app (e.g., DroneDeploy, Pix4Dcapture)
- A photogrammetry processing platform (e.g., Pix4Dmatic, Propeller Aero)
- An AI change detection tool (e.g., DroneDeploy's Change Detection, or custom using OpenAI GPT-4 Vision)
- A project management dashboard (e.g., Procore, Autodesk BIM 360)

## Step 1: Plan and Execute Drone Flights

1. **Set up flight plan** using DroneDeploy or Pix4Dcapture. Define the area of interest (AOI) with overlap >75% for accurate 3D reconstruction.
2. **Fly the drone** at consistent altitude (e.g., 100-120m AGL) and same time of day to minimize shadows.
3. **Capture images** with geotags. Ensure sufficient ground control points (GCPs) for accuracy.
4. **Upload images** to cloud processing platform (DroneDeploy or Pix4Dmatic).

## Step 2: Process Orthomosaic Maps

1. **Create a new project** in Pix4Dmatic or Propeller Aero.
2. **Import images** and run photogrammetry processing. Typical processing time: 1-4 hours depending on area size.
3. **Generate orthomosaic** (georeferenced 2D map) and digital surface model (DSM).
4. **Export** as GeoTIFF or cloud-optimized format.

## Step 3: Run AI Change Detection

1. **Load baseline orthomosaic** (e.g., from previous week) and current orthomosaic into DroneDeploy's Change Detection tool.
2. **Configure detection parameters**: set minimum change area (e.g., 1m²), ignore vegetation changes, and select layers (e.g., elevation, RGB).
3. **Run analysis** – AI identifies areas of excavation, new structures, material stockpile changes, etc.
4. **Review results** – overlay change heatmap on current map. Export as KML or PDF report.

**Alternative AI approach using GPT-4 Vision:**
- Use Python script to tile orthomosaics and send image pairs to GPT-4 Vision API with prompt: "Identify all construction changes between these two aerial images. List coordinates and describe each change."
- Parse JSON response and map changes onto a web dashboard.

## Step 4: Integrate with Project Management

1. **Export change report** (PDF, CSV, or API) from DroneDeploy or custom script.
2. **Upload to Procore** or Autodesk BIM 360 as a new issue or progress photo set.
3. **Set up automated notifications** (e.g., via Zapier) to email stakeholders when changes exceed threshold.
4. **Update schedule** – link detected changes to specific work packages (e.g., "Foundation pour completed").

## Step 5: Automate and Scale

1. **Schedule recurring flights** using DroneDeploy's auto-flight feature (e.g., daily at 10 AM).
2. **Use webhooks** to trigger processing and change detection automatically after upload.
3. **Create a dashboard** in Power BI or Tableau that pulls data from DroneDeploy API and Procore API for real-time progress visualization.

## Best Practices

- **Consistent lighting**: Fly within 2 hours of solar noon.
- **Ground control**: Use at least 5 GCPs for sub-2cm accuracy.
- **Data storage**: Keep all orthomosaics in cloud storage (e.g., AWS S3) with versioning.
- **Compliance**: Ensure drone pilot has Part 107 license (US) or equivalent.

## Troubleshooting

- **Poor alignment**: Increase image overlap to 80% and add more GCPs.
- **False positives in change detection**: Adjust sensitivity and mask out known non-construction areas (e.g., parked cars).
- **Processing delays**: Use GPU-accelerated cloud processing (Pix4Dmatic offers this).

## Conclusion

This workflow transforms raw drone imagery into actionable progress insights. By automating change detection, you reduce manual inspection time by 80% and improve accuracy. Start with a pilot project on a small site, then scale to multiple projects.
